## Important Findings

### 2. @-mention stores UUID instead of display name — holdout caught it

**Root Cause:** agent_limitation
**Pattern:** new
**Destination:** spec amendment

Tiptap Mention suggestion items use `{ id: n.id, title: n.title }` but the extension expects `{ id, label }`. The `title` field isn't mapped to `label`, so mention nodes store `{ id: UUID, label: undefined }` and render UUIDs. The holdout criterion (REQ-015) caught this — visible acceptance criteria did not.

**Impact:** @-mentions display as raw UUIDs. Core note-linking feature broken.
**Proposed Fix:** Change suggestion item shape from `{ id: n.id, title: n.title }` to `{ id: n.id, label: n.title }` in TiptapNoteEditor.tsx. Update the renderer to use `item.label`. Add explicit acceptance criterion: "mention nodes store `{ id, label }` verified by reading ProseMirror JSON after save."
